===============================================================
KNOWN AI CHECK MISTAKES — READ BEFORE MAKING ANY JUDGEMENT
These are real cases where the AI previously checked incorrectly.
Use them to avoid the same errors.
===============================================================

------------------------------------------
BENDING & SCHEDULE CHECKS
------------------------------------------

[CHECK: Last Position Number vs Title Block]

[MISTAKE] Last Position Number — Falsely flagging when 100+ series bars are present
  WRONG: Reporting mismatch for "letzte Stabstahlposition" because the Stabliste contains Pos 100, 101, 102, ... in addition to the regular bars.
  CORRECT: "letzte Stabstahlposition" refers ONLY to the highest Pos number ≤ 99. Pos 100+ is a separate special accessory series and does NOT count. If the title block value equals the highest regular Pos (≤ 99), check PASSES.

[MISTAKE] Last Position Number — Comparing values across different sheets or reference images
  WRONG: Flagging a mismatch because 'letzte Stabstahlposition = 10' on the FT-Brandwand reference image, while the actual drawing shows 19.
  CORRECT: Read ONLY from the submitted PDF drawing. Compare the Stabliste and title block on THE SAME SHEET. Never compare values from reference/example images or across different drawing sheets.

[CHECK: Pos Coverage]

[MISTAKE] Pos Coverage — False "missing schema" for bars annotated in section margins
  WRONG: Flagging Pos 4, 11, 17, 100, 101, 102, 103, 104 as missing schemas when they are clearly annotated in the margins of Schnitt a-a, Schnitt b-b, or Schnitt c-c.
  CORRECT: Any bar annotation of the form "n ø d L=xxx cm" or "n ø d/spacing" shown anywhere in the drawing (section margins, Bewehrung plan, detail areas) counts as full coverage for that Pos. A dedicated schema box is NOT required. Spacer/mesh bars annotated as "n ø d -M.E." in section views are also covered.

[MISTAKE] Pos Coverage — False "missing schema" for Pos 100+ bars
  WRONG: Flagging Pos 101, 102, 103, 104 because no separate bending schema box exists.
  CORRECT: Pos 100+ bars are often shown as simple bar sketches "n ø d L=xxx cm" in the margins of Schnitt b-b or c-c. These count as full coverage.

[CHECK: Total Mass Arithmetic]

[MISTAKE] Total Mass Arithmetic — Flagging small rounding differences
  WRONG: Flagging a 0.70% deviation (2.75 kg out of 392 kg) as an error.
  CORRECT: Only flag if the difference is genuinely significant. Small deviations from PDF reading precision are expected.

[MISTAKE] Total Mass Arithmetic — Reading values from reference images instead of the PDF drawing
  WRONG: Flagging mass_arithmetic because a table footer on the FT-Brandwand reference image sheet is illegible, while the actual PDF drawing sheet passes.
  CORRECT: Read ONLY from the submitted PDF drawing. Any table or title block visible in reference/example images must be completely ignored. Only values in the actual drawing PDF are valid. If a table footer is illegible on the PDF, skip that table — do not flag it.

[CHECK: Bending Angle / Mandrel Diameter]

[MISTAKE] Bending Angle — Misidentifying bar diameter
  WRONG: Seeing dbr = 3.2 cm in Schnitt a-a and assuming the bar is Ø10 (min. dbr for Ø10 = 4.0 cm → false FAIL reported).
  CORRECT: The bar diameter MUST be read explicitly from the schema label or Stabliste row. In that case the bar was Ø8 (min. dbr for Ø8 = 3.2 cm → PASS). Never guess or infer the bar diameter. If unclear, do NOT flag.

[MISTAKE] Bending Angle / Bar Length / Mass Arithmetic — Outputting PASS narrative as a finding
  WRONG: Returning an item in the issues list that describes a passing result, e.g. 'dbr=6.4 cm exactly meets the minimum — PASS', 'No bar length mismatches found', or 'computed total matches footer — PASS'. Python code counts every item as a failure.
  CORRECT: Output NOTHING in the issues list when a check passes. An empty issues list is the correct response when all checks pass. Only output items that represent actual violations.


------------------------------------------
SPELLING & TITLE BLOCK CHECKS
------------------------------------------

[CHECK: Spelling]

[MISTAKE] Spelling — Flagging PDF font rendering artifacts as spelling errors
  WRONG: Flagging 'FACHPLANFR/PLANFRSTFLLER' as a spelling error when the PDF actually shows 'FACHPLANER/PLANERSTELLER', or flagging 'MI I-Nummer' instead of 'MTT-Nummer'. These are PDF rendering artifacts from compressed or low-resolution fonts.
  CORRECT: Do NOT flag text that appears to have missing or distorted characters due to PDF rendering. Only flag when the spelling error is unambiguous and clearly a human typing mistake. If the text looks like a rendering artifact (compressed letters, partially missing characters), assume the printed drawing is correct and skip it.

[CHECK: Section Name Completeness]

[MISTAKE] Section Name Completeness — Requiring "Schnitt X-X" specifically
  WRONG: Flagging section cut "1-1" as missing because only "Draufsicht 1-1" is present, not a vertical "Schnitt 1-1".
  CORRECT: Any view carrying the same designation number satisfies the requirement. "Draufsicht 1-1", "Schnitt 1-1", "Detail 1-1" — all count equally. Only flag if NO view of any type with that number exists on the sheet.

[CHECK: Scale Consistency]

[MISTAKE] Scale Consistency — Flagging colon-as-period PDF rendering as a scale error
  WRONG: Flagging '1.25 1.200' as incorrect scale notation when the drawing actually shows '1:25 1:200'. The period is a PDF rendering artifact where the colon character is rendered incorrectly.
  CORRECT: In some PDF fonts, the colon (:) in scale notation such as '1:25' or '1:200' may render as a period ('1.25', '1.200'). This is a rendering artifact, NOT an error. Only flag if the numeric ratio itself clearly differs (e.g. 1:25 vs 1:50).

[CHECK: Parts Quantities]

[MISTAKE] Parts Label Consistency — Wrong NOT FOUND condition
  WRONG: Marking parts_quantities as NOT FOUND just because one of the two tables (Einbauteilliste or Montageteilliste) is absent. A drawing does not need both tables.
  CORRECT: Only mark parts_quantities as NOT FOUND when NEITHER Einbauteilliste NOR Montageteilliste is present on the sheet. If at least one table is present, perform the check against whichever table(s) exist. Cross-reference part labels visible in views only against the tables that are actually on the sheet.

[CHECK: 3D View]

[MISTAKE] 3D View — Failing to recognize a clearly labeled "3D Perspektive" view
  WRONG: Reporting "No 3D isometric or perspective view present on sheet" when the sheet contains a view labeled "3D Perspektive M 1:25".
  CORRECT: Any view labeled "3D Perspektive", "Perspektive", "3D Ansicht", or containing "3D" or "Perspektiv" in the title IS a valid 3D view — check PASSES immediately. The visual rendering style (line drawing vs shaded) is irrelevant.

[MISTAKE] 3D View — Failing to recognize line-drawing axonometric views
  WRONG: Reporting "No 3D perspective, isometric, or axonometric view of the wall element is present on the sheet" when a clear engineering isometric/axonometric line drawing of the wall is visible on the sheet (even without a label or title).
  CORRECT: The standard 3D view in precast wall drawings is an UNLABELED axonometric line drawing placed in a corner of the sheet. It shows the entire wall element tilted at ~45° so the viewer sees the front face AND one or more side/top edges at the same time — giving a complete "overview" impression of the 3D wall body. Typical appearance: tall rectangular slab shown diagonally, with embedded hardware (anchors, bolts, rebar tails) visible on the face, drawn in pure linework. This view does NOT need: a title, a "3D" label, shading, fill, or a scale. HOW TO SCAN: look at ALL corners and margins. If any region shows a wall-shaped rectangular outline drawn at a diagonal angle (clearly not a plan or elevation), that IS the 3D view → check PASSES immediately.


------------------------------------------
REBAR LABEL & DIMENSION CHECKS
------------------------------------------

[CHECK: Spacer/Clamp Label Suffix]

[MISTAKE] Spacer/Clamp Label (-M.E.) — Checking wrong bars
  WRONG: Flagging ALL bar positions for missing "-M.E." suffix.
  CORRECT: ONLY bars that have already been seen labeled with "-M.E." somewhere on the sheet are spacer/clamp bars. Identify those Pos numbers first, then check that ALL other labels for those same Pos numbers also carry "-M.E.". Never check a bar that was never labeled "-M.E." anywhere.

[MISTAKE] Spacer/Clamp Label — Flagging bending schema label for missing -M.E.
  WRONG: Flagging the bending schema label '17 42 ø 8 L=43cm' for missing '-M.E.' suffix. Bending schemas show bar geometry and length — they do not carry the '-M.E.' suffix.
  CORRECT: The '-M.E.' suffix check applies ONLY to section view callouts (Schnitt annotations). Bending schema figures and Stabliste rows do NOT need '-M.E.' — never flag them.

[MISTAKE] Spacer/Clamp Label — Flagging truncated or cut-off text
  WRONG: Flagging 'ø 8 /4' (truncated text in a tight area of the drawing) as missing '-M.E.' suffix.
  CORRECT: Truncated or partially visible labels are PDF rendering issues. Do NOT flag text that appears cut off or incomplete — the full label on the printed drawing likely contains '-M.E.'.

[CHECK: Vertical Pin Width]

[MISTAKE] Vertical Pin Width — Checking wrong Pos with oversized width
  WRONG: Flagging Pos 4 schema showing width=35cm as a vertical pin error when required width is 25cm. Pos 4 is not a vertical pin — it is a different bar type (structural tie or hairpin).
  CORRECT: Only check schemas in the SIDE section (Schnitt a-a) whose labeled width is close to (wall_width – 2×Cv). If a schema's labeled width is much larger than the required value (e.g. 35cm when required is 25cm), it is not a vertical pin — skip it.

[MISTAKE] Pin/Spacer width checks — NOT FOUND reported when dimensions are visible on sheet
  WRONG: Adding pin_width_vertical, pin_width_horizontal, or spacer_width to not_found even though wall_width and Cv are clearly visible on the drawing.
  CORRECT: WHERE TO FIND EACH DIMENSION: wall_width: Look in the FORMWORK cross-section views (Draufsicht X-X, Schnitt X-X in the Ansicht/formwork area). A dimension line spanning the full wall thickness shows the value in cm (e.g. "30" = 30 cm). Cv:         Read from the title block "BETONDECKUNG" table, column labeled "Cv" or "Cᵥ". Value is in mm — divide by 10 for cm. E.g. "25" → Cv = 2.5 cm. Do NOT use Cmin,dur or ΔCdev. Ø_layer1:   Read from Schnitt a-a in the Bewehrung. Take the FIRST rebar layer counting from the outer wall face inward. Its label (e.g. "ø 12/15") gives the diameter. E.g. ø12 → Ø_layer1 = 1.2 cm.

[MISTAKE] Vertical vs Horizontal Pin identification
  WRONG: Confusing vertical and horizontal pins, leading to wrong width formula applied.
  CORRECT: - Vertical pins (stirrups): schematized in the SIDE section view (Schnitt a-a style), appear as narrow U-shapes or tall rectangles. Formula: wall_width – 2×Cv. - Horizontal pins: schematized in the BOTTOM cross-section view (Schnitt b-b style), appear as wide flat rectangles. Formula: wall_width – 2×Cv – 2×Ø_layer1. Read the pin schema from the correct section before applying the formula.

[MISTAKE] Pin/Spacer Width — Using example numbers from prompt instead of actual drawing values
  WRONG: Calculating pin or spacer width using illustrative values from the prompt or knowledge base (e.g. wall_width=30, Cv=2.5, Ø_layer1=1.2) instead of the actual values found in this drawing. This causes false FAILs when the real wall has different dimensions.
  CORRECT: ALWAYS read wall_width, Cv, and Ø_layer1 directly from the drawing being analyzed — these values differ on every drawing. (1) wall_width: read the dimension line on the formwork cross-section (Draufsicht / Ansicht Schnitt). (2) Cv: read from the BETONDECKUNG table in the title block, Cv column, divide mm by 10. (3) Ø_layer1: read the outermost rebar label in Schnitt a-a. Only after recording these actual values apply the formula. If a value cannot be found in this drawing, add the check to not_found — never substitute an assumed or example value.

[CHECK: Horizontal Pin Width]

[MISTAKE] Horizontal Pin Width — Checking wrong Pos with oversized dimension
  WRONG: Flagging Pos 50 schema showing a dimension of 105cm as a horizontal pin error when required width is ~22cm. The 105cm value is the total bar length or leg dimension, not the stirrup width.
  CORRECT: Only check schemas in the HORIZONTAL section (Schnitt b-b) whose SHORT dimension is close to (wall_width – 2×Cv – 2×Ø_layer1). If a labeled dimension is several times larger than the expected pin width (~22cm), it is not a horizontal pin — skip it entirely.

[CHECK: Spacer/Clamp Width]

[MISTAKE] Spacer/Clamp Width — Checking non-spacer positions
  WRONG: Checking Pos 4 for spacer width when Pos 4 has no '-M.E.' label anywhere on the drawing.
  CORRECT: Only check Pos numbers that carry '-M.E.' somewhere on the drawing. Pos without '-M.E.' are regular rebar, not spacers — skip them entirely for the spacer_width check.
